Prompt consigliato per Cursor (Agent) — validazione link + coerenza titoli Supabase
=====================================================================================

1) Validazione HTTP a lotti (50 bandi per volta fino alla fine):

   cd /percorso/Raas-automazioni
   bash tools/run-validate-bandi-supabase-batches.sh

   (Opzionale: STEP=100 CONC=20 …). Report in data/bandi-validate-batches/

2) Analisi JSON per http_unreachable: dove possibile proponi URL istituzionali.

3) Coerenza titolo (database) vs contenuto pagina (<title> + parole nel path URL):

   bash tools/run-audit-coherence-bandi-supabase-batches.sh

   (lotti da STEP=200; oppure singolo lotto: node tools/audit-bandi-coherence-supabase.js --offset=0 --limit=200)

4) Se ci sono incongruenze evidenti tra titolo salvato e pagina linkata, e l’utente ha
   configurato la service role, disattiva i record errati (attivo=false):

   export SUPABASE_SERVICE_ROLE_KEY="…"
   bash tools/run-audit-coherence-bandi-supabase-batches.sh --deactivate

   (oppure: node tools/audit-bandi-coherence-supabase.js --deactivate-incongruent --limit=200 …)

5) Non applicare PATCH su Supabase senza SUPABASE_SERVICE_ROLE_KEY.

6) Non chiedere conferma tra un lotto e l’altro salvo errori bloccanti (rete, 401/403 REST).

🔍 I Tre Core Web Vitals Spiegati

LCP - Largest Contentful Paint

Misura il tempo per visualizzare l'elemento più grande del viewport. Google considera "buono" un LCP sotto 2.5 secondi. I siti RaaS raggiungono 0.4-0.8 secondi. Le cause di LCP lento: immagini non ottimizzate, font ritardati e blocking JavaScript.

INP - Interaction to Next Paint

Ha sostituito il FID nel marzo 2024. Misura la reattività complessiva alle interazioni (soglia: sotto 200ms). Molti siti WordPress superano i 500ms a causa di JavaScript pesante.

CLS - Cumulative Layout Shift

Misura gli spostamenti imprevisti durante il caricamento. Soglia ideale: sotto 0.1. Colpevoli comuni: immagini senza dimensioni, banner caricati in ritardo, font che cambiano layout.

🛠️ Strategie di Ottimizzazione: Da 50 a 90+

1. Critical CSS Inline

La tecnica più efficace: inserire nel <head> il CSS necessario per il contenuto above-the-fold. Da sola può migliorare il punteggio di 15-25 punti.

<!-- CSS critico inline --> <style> body { font-family: 'Plus Jakarta Sans', sans-serif; } .hero { background: #0a2540; padding: 80px 24px; } </style> <!-- Resto CSS asincrono --> <link rel="preload" href="style.css" as="style" onload="this.rel='stylesheet'">

2. Ottimizzazione Immagini

Le immagini pesano il 50-70% della pagina. Strategia: WebP/AVIF (-30-50% vs JPEG), width e height sempre specificati, lazy loading sotto la piega.

<img src="hero.webp" alt="Descrizione" width="1200" height="600" loading="lazy" decoding="async">

3. Eliminazione Render-Blocking

Ogni file CSS/JS non critico nel <head> blocca il rendering. WordPress ne carica 15-30. La soluzione: differire tutto ciò che non è essenziale per la visualizzazione iniziale.

4. Preconnect e DNS-Prefetch

Se carichi risorse esterne (Google Fonts, Analytics), il preconnect avvia la connessione in anticipo, risparmiando 100-300ms per dominio.

5. Font Optimization

Font-display: swap + preload dei font critici. Limitare varianti di peso e stile.

❌ I 7 Errori Più Comuni

Errori da Evitare

🔴
Immagini PNG/JPG non compresse — Usa WebP/AVIF, dimensioni al viewport necessario.
🔴
Troppi plugin WordPress — Ogni plugin aggiunge ~150ms al caricamento.
🔴
Hosting economico condiviso — Il TTFB dipende dal server.
🔴
Video autoplay non ottimizzati — Possono aggiungere 5-10MB.
🔴
Google Fonts sincroni — Bloccano rendering per 300-500ms.
🔴
Immagini senza dimensioni — Causano layout shift.
🔴
Script terze parti non differiti — Analytics, pixel, chat possono aggiungere 1-3s.

❓ FAQ: 10 Domande sul PageSpeed

Cos'è il PageSpeed Score?

Punteggio da 0 a 100 di Google che misura velocità ed esperienza utente. Sopra 90 è "buono". Influenza ranking e conversioni.

Quali sono i Core Web Vitals 2026?

LCP (velocità caricamento, ideale <2.5s), INP (reattività, ideale <200ms), CLS (stabilità layout, ideale <0.1).

Il PageSpeed impatta sulle vendite?

Sì: ogni secondo di ritardo riduce le conversioni del 7% secondo Google.

WordPress può raggiungere PageSpeed 90?

Molto difficile. La maggior parte si ferma tra 50 e 70 su mobile.

Cosa sono LCP, INP e CLS?

Il lazy loading aiuta?

Sì, riduce il peso iniziale caricando immagini solo quando visibili.

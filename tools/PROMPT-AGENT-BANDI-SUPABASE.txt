Prompt consigliato per Cursor (Agent) — validazione link + coerenza titoli Supabase
=====================================================================================

1) Validazione HTTP a lotti (50 bandi per volta fino alla fine):

   cd /percorso/Raas-automazioni
   bash tools/run-validate-bandi-supabase-batches.sh

   (Opzionale: STEP=100 CONC=20 …). Report in data/bandi-validate-batches/

2) Analisi JSON per http_unreachable: dove possibile proponi URL istituzionali.

3) Coerenza titolo (database) vs contenuto pagina (<title> + parole nel path URL):

   bash tools/run-audit-coherence-bandi-supabase-batches.sh

   (lotti da STEP=200; oppure singolo lotto: node tools/audit-bandi-coherence-supabase.js --offset=0 --limit=200)

4) Se ci sono incongruenze evidenti tra titolo salvato e pagina linkata, e l’utente ha
   configurato la service role, disattiva i record errati (attivo=false):

   export SUPABASE_SERVICE_ROLE_KEY="…"
   bash tools/run-audit-coherence-bandi-supabase-batches.sh --deactivate

   (oppure: node tools/audit-bandi-coherence-supabase.js --deactivate-incongruent --limit=200 …)

5) Non applicare PATCH su Supabase senza SUPABASE_SERVICE_ROLE_KEY.

6) Non chiedere conferma tra un lotto e l’altro salvo errori bloccanti (rete, 401/403 REST).

7) Schema URL (definizione unica):
   - Tabella Supabase: public.bandi — colonna canonica del link pubblico: **url** (testo).
   - Legacy: alcuni record possono avere ancora **link**; in lettura il codice usa url || link.
   - File locale data/bandi.json: campo **url_bando** (fallback se il sito non usa Supabase).

8) Validazione gratuita sul JSON locale (nessun secret):
   node tools/validate-bandi-links-free.js [--limit=N]
   Policy aggregatori: tools/bandi-link-policy.js

9) Audit automatico venerdì ~07:00 CET (GitHub Actions):
   .github/workflows/weekly-audit.yml esegue anche la sezione “Validazione link data/bandi.json”
   (stesso script del punto 8). Per il database in produzione usare i punti 1 e 3 con le variabili d’ambiente.

10) Regole progetto per l’agente: .cursor/rules/bandi-supabase.mdc (globs tools/bandi*, data/bandi.json, bandi.html, admin.html).

11) Statistiche keyword per suggerimenti ricerca (batch, niente GPU):
   npm run build-bandi-keywords
   (oppure node tools/build-bandi-keyword-stats.js — opzioni --source=supabase|json|auto, --out=…)
   Genera data/bandi-keyword-stats.json; bandi.html lo usa per il datalist della ricerca.

12) RLS + controllo lettura anon:
   supabase/migrations/20260410120000_bandi_rls_anon_access.sql (applicare su progetto Supabase se serve).
   npm run verify-bandi-public-read — deve stampare PASS (bandi attivi via chiave anon).
